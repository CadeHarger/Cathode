## When to Use

- Use this skill only when invoked by the user

## Instructions

Reflect upon the YAML schema documentation you used in this thread (`backend/backend_schema.yaml` and/or `cathode-frontend/frontend_schema.yaml`). Were the parts you paid most attention to helpful? What could have been mentioned that wasn't and would have saved tokens? Can the relevant section be more concise?

Reflect and then update the appropriate schema file. Beware of documentation bloat — only mention what is necessary. Mentioning every detail helps rare deep changes but wastes tokens in most tasks.
